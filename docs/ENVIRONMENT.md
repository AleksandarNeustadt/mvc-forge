# Environment Reference

Runtime configuration lives in `app/.env`. Start from `app/.env.example` and never commit production secrets.

## Bootstrap Override

If `app/` is not located next to `public_html/`, set `APP_ROOT_OVERRIDE` in server config before PHP bootstrap starts.

Apache:

```apache
SetEnv APP_ROOT_OVERRIDE /absolute/path/to/app
```

php-fpm:

```ini
env[APP_ROOT_OVERRIDE] = /absolute/path/to/app
```

## Application Variables

- `APP_DEBUG` - enable debug mode only in local development
- `APP_LOG_LEVEL` - minimum log level
- `APP_LOG_ALERT_EMAIL` - optional alert recipient
- `APP_LOG_ALERT_THROTTLE_SECONDS` - alert throttling window
- `APP_URL` - canonical application base URL
- `APP_KEY` - application secret generated by `php app/bin/console app:key`
- `VIEW_CACHE` - enable compiled view cache
- `CORS_ALLOWED_ORIGINS` - comma-separated allowed origins
- `CORS_ALLOW_CREDENTIALS` - whether browser credentials are allowed in CORS responses

## Database Variables

- `DB_CONNECTION` - currently expected to be `mysql`
- `DB_HOST`
- `DB_PORT`
- `DB_DATABASE`
- `DB_USERNAME`
- `DB_PASSWORD`
- `DB_CHARSET`

## Session and Branding

- `SESSION_LIFETIME` - session lifetime in minutes
- `BRAND_NAME` - site brand displayed in UI and emails; `install:setup --site-name=...` writes this value
- `BRAND_TAGLINE` - fallback title/tagline

## Local Frontend Dev

- `VITE_DEV_SERVER_ORIGIN` - optional Vite HMR origin when `APP_DEBUG=true`
